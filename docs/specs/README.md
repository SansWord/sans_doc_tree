# Historical tier — specs

Per-milestone specs and brainstorms. **Allowed to go stale.** Each file is a snapshot of the thinking
at the time — *how we got here* — and is **not** the source of truth.

- Don't cite these where they conflict with the code or the root `CLAUDE.md`.
- **Migrate before you retire:** before treating a milestone as "done," move any still-current
  decision that lives *only* here into a maintained `docs/*.md` first.
- **Kept, never deleted.** "Allowed to go stale" means only that the *content* may drift from the
  code over time — the files themselves stay forever as your record of *why* decisions were made.
  This is your product-decision history; don't delete it.

Add new milestone spec files alongside this README (e.g. `2026-06-feature-x.md`); leave the old
ones in place.
